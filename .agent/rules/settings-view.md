# Settings View

## Module Structure

```
views/settings/
├── mod.rs              — State, Message, Action, update; `SettingsTab` enum (6 variants), `route_write` helper
├── entries.rs          — Entry building / filtering / cross-tab search
├── items.rs            — Re-exports `SettingValue` / `SettingMeta` / `SettingItem` / `SettingsEntry` from the data crate, plus the `MacroRows::{new, take, finish}` helper used by the per-tab builders. Every builder ends with `macro_rows.finish()` (debug-panics naming any un-taken key; warns in release); conditional rows follow the take-unconditionally-push-conditionally convention so consumption is data-independent. Construct `SettingMeta` via `SettingMeta::new(key, label, category).with_subtitle(...)`
├── items_general.rs    — Library, Display, Behavior, Window & Tray, Advanced, Account
├── items_interface.rs  — Navigation, Slot List, Player Bar, Font, Metadata Strip, Artwork Overlays, Artwork Column
├── items_playback.rs   — Transitions, Volume Normalization, Scrobbling, Rating Reminder, Playlists
├── items_hotkeys.rs    — Per-category hotkey entries
├── items_theme.rs      — Mode, Display, Select Theme, Background / Foreground / Accent / Semantic colors, Chrome Border
├── items_visualizer.rs — Frame, Signal, Bars, Bar Colors (Dark/Light), Lines
├── sub_lists.rs        — `SubListState` (color gradient editor — in-place detail-pane replacement) + `FontSubListState` (font picker — modal overlay)
├── presets.rs          — Theme discovery + apply
├── rendering.rs        — Slot rendering: detail-row chrome, headers, items, colors, hotkey badges, toggle sets
├── sentinel.rs         — `SentinelKind` enum for stringly-typed pseudo-keys (restore-all, restore-group, apply-preset). Routes through `route_write` instead of `update_config_value`
├── test_support.rs     — `#[cfg(test)]`-only helpers (`extract_keys`, `header_labels`, `section_slice`, `palette_prefix_from`, `assert_section_keys`) shared by the items/items_theme test modules. Structure tests pin section headers + per-section ordered KEY lists (failures name the diverging key), keeping at most one coarse item-count backstop per tab
└── view.rs             — Persistent two-pane layout: 340 px categories sidebar + scrollable detail pane (or narrow-variant horizontal chip strip below `NARROW_BREAKPOINT_WIDTH = 1400 px`)
```

Settings dispatch tables for **General / Interface / Playback** are generated by `define_settings!` (`data/src/services/settings_tables/`) — the macro emits both the dispatch arm and the `dump_<tab>_tab_player_settings` round-trip, so adding a settable knob means adding one entry there rather than touching the legacy match arm. Theme / Visualizer / Hotkey items still live in their items_*.rs files.

## Architecture

- 6 tabs in display order: **General, Interface, Playback, Hotkeys, Theme, Visualizer**
- Persistent sidebar drives `active_category: SettingsTab`; the detail pane renders `cached_entries` **verbatim** — `SettingsPage::view(window_width, window_height)` takes only the two window dims and never rebuilds entries. The update path owns all rebuilds via `refresh_entries` (`build_category_sections` / `search_all_entries`) and `Nokkvi::refresh_settings_entries_if_dirty` (gated on `current_view == Settings`, fires when `config_dirty || cached_entries.is_empty()`). Rebuild triggers: view entry (`handle_switch_view`), tab switch, search edits, every `SettingsAction` write (tail of `handle_settings`), theme/visualizer/settings hot-reloads, hotkey rebind, the chrome light-mode + crossfade toggles, the `SetLightModeAtomic` side-effect arm in `dispatch_settings_side_effect` (must rebuild AFTER the atomic flips — `handle_player_settings_loaded` runs earlier in the same dispatch and rebuilds against the old mode), the reset-visualizer dialog, the Local Music Path dialog commit (`update/text_input_dialog.rs`), the default-playlist picker, and `handle_player_settings_loaded`. Two writers deliberately mark dirty WITHOUT refreshing because they cannot co-occur with an open Settings view: the artwork-vertical-height drag (`handle_artwork_vertical_drag`, runs in a library view) and the Playlists-view "Set as Default Playlist" action. **Any new writer of settings-visible state must set `settings_page.config_dirty = true` and (if it can run while Settings is open) call `refresh_settings_entries_if_dirty()`** — there is no per-frame fallback to mask a missed mark.
- Sidebar nav: `SidebarUp` / `SidebarDown` (Shift+Backspace / Shift+Tab by default) move the sidebar cursor; `SidebarClickItem(idx)` jumps. The sidebar is a fixed six-row column rather than a slot list — six rows always fit, so `sidebar_slot_list.viewport_offset` is reused only as the cursor index.
- Narrow variant: when `window_width < NARROW_BREAKPOINT_WIDTH` (1400 px) the layout collapses to a horizontal chip strip + search header above the detail pane.
- Search: lives in the sidebar header; cross-tab search combines entries from every tab and a tab-name match pulls the whole tab in.
- Sub-lists: the color-gradient editor replaces the detail pane in place via `SubListState`; the font picker stacks as a modal via `FontSubListState`.

## SettingValue Types

| Type | Interaction |
|------|-------------|
| `Float` / `Int` | ←/→ adjust with clickable arrow buttons |
| `Bool` | Toggle via clickable On/Off badges |
| `Enum` | Cycle via clickable option badges (`EditSetValue`) |
| `Text` | Free-text input row (used for `font_family`, custom library path, etc.) |
| `ToggleSet` | Multi-select badges. ←/→ moves cursor, Enter toggles, ↑/↓ sets on/off |
| `HexColor` | Direct hex input. `HEX_EDITOR_INPUT_ID` + `FocusHexInput` action for auto-focus |
| `ColorArray` | Opens a sub-slot-list for gradient editing |
| `Hotkey` | Badge display + key capture mode (Esc cancels, Delete resets, steal-on-conflict) |

## Key Patterns

- **Typed config write routing** (`src/config_writer.rs`):
  - `ConfigKey::AppScalar` → `config.toml`
  - `ConfigKey::Theme` / `ThemeArrayEntry` → active theme file
  - The dispatch handler matches on the variant — never sniff key prefixes
  - Routes through `route_write` (helper at the top of `views/settings/mod.rs`) which short-circuits `general.*` keys to `WriteGeneralSetting` and forks the remainder into `WriteConfig { ConfigKey::Theme | AppScalar, ... }`; sentinel pseudo-keys (`__action_logout`, `__restore_theme`, `__preset_<n>`, etc.) use `SentinelKind` (in `views/settings/sentinel.rs`) instead of stringly keys
- **`verbose_config` toggle**: combined persist + TOML rewrite in a single async task to avoid races
- **Strip visibility toggles** (Metadata Strip section): `ToggleSetToggle` flips the cached entry then emits `WriteGeneralSetting`. The mini-player column (`TrackInfoDisplay::MiniPlayer`) paints its own colored title/artist/album text widgets; the per-segment color/kind plumbing the old progress-bar overlay used has been removed.
- **Artwork Column** (Interface tab): mode (`Auto` / `AlwaysNative` / `AlwaysStretched` / `AlwaysVerticalNative` / `AlwaysVerticalStretched` / `Never`) + stretch fit + draggable width pct. Width drag persists via `WriteGeneralSetting`
- **System Tray** (General tab): `show_tray_icon` + `close_to_tray`. Live-toggle starts/stops the StatusNotifierItem subscription without restart
- **Font picker**: modal overlay (`FontSubListState`, stacked above the detail pane). System fonts via `font-kit`, `LazyLock`-cached
- **Theme picker**: a list of `SentinelKind::PresetTheme(i)` rows rendered inline in the Theme tab's detail pane (`build_theme_items`). Activating a row rewrites `theme = "name"` in `config.toml` and triggers hot-reload; the active theme gets a "● active" suffix
- **Search pitfall**: `SlotListDown` must navigate within `cached_entries` — only `SearchChanged` rebuilds entries
- **Icons**: `SettingItem` and `SettingsEntry::Header` carry an icon path. SVGs in `assets/icons/` are auto-registered by `build.rs`; unknown paths fall back to `play.svg` with a warn log
